## Open Questions / Ambiguities Found
- Readiness scoring: worksheet totals each dimension separately with &lt;4 / ≥4; codebase does that **and** requires all three Met to Accept (`allMet`). Whether governance intends Accept-on-partial (e.g. Kaizen when Not Met) is ambiguous relative to the diagram vs the sheet — classified Divergent on 3.3/3.4 rather than guessed.
- Diagram Review Accept says “INCLUDE APPLICABLE TOOL STACK” at Accept; codebase defers stack to post-qualification Tool Selection. Unclear if diagram means mandatory at Accept or only “applicable later.”
- Assisted “session history” / one-session-at-a-time: Profile copy acknowledges a future feature; no mechanism found to audit beyond drafts.
- HOW guide question maps to data/systems in code vs “proposed AI tool and solution” on the diagram — field intent vs label mismatch.
- Section C “End of Assessment” when exclusion applies: UI encourages Not Qualified path but does not auto-close assessment or set a distinct non-AI status.
- Cancel on review: status `Cancelled` is supported; mock email alert on cancel is Not present (unlike Not Qualified).
- Primary/Supporting checklist labels are paraphrased (shorter) vs spreadsheet verbatim; treated as Supported for structure with that note.
- Diagram Review criteria block says “Visibility”; sheet/code use “Viability” — documentation inconsistency noted in prompt; code aligns with sheet.
